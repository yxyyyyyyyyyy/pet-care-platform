# 宠物健康养护平台 - 部署指南

## 概述

本项目采用前后端分离架构，后端部署在 Render（免费层），前端部署在 Vercel（免费层）。

## 一、后端部署（Render）

### 1. 创建 Render 账户

访问 [Render](https://render.com)，使用 GitHub 账号注册登录。

### 2. 创建 Web Service

1. 点击 Dashboard 右上角 **"New"** → **"Web Service"**
2. 选择 **"Build and deploy from a Git repository"**
3. 连接你的 GitHub 仓库，选择项目分支

### 3. 配置部署参数

| 配置项 | 值 |
|--------|------|
| Name | pet-care-backend |
| Region | Singapore（亚洲区） |
| Branch | main（或你的分支名） |
| Root Directory | backend |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `gunicorn app:app` |
| Instance Type | Starter（免费） |

### 4. 配置环境变量

在 **"Environment"** 标签页添加以下环境变量：

| 变量名 | 值 | 说明 |
|--------|------|------|
| SECRET_KEY | 自动生成或自定义 | Flask 加密密钥 |
| JWT_SECRET_KEY | 自动生成或自定义 | JWT 签名密钥 |
| ALLOWED_ORIGINS | `https://your-vercel-domain.vercel.app` | 前端 Vercel 域名 |
| PYTHON_VERSION | `3.11.0` | Python 版本 |

### 5. 部署

点击 **"Create Web Service"** 开始部署，等待构建完成。

部署成功后，你将获得一个公网地址，格式类似：
`https://pet-care-backend-xxxx.onrender.com`

### 6. 验证后端

访问 `https://your-backend-domain.onrender.com/api/health`，返回 `{"status": "ok"}` 表示部署成功。

---

## 二、前端部署（Vercel）

### 1. 创建 Vercel 账户

访问 [Vercel](https://vercel.com)，使用 GitHub 账号注册登录。

### 2. 创建 Project

1. 点击 **"Add New"** → **"Project"**
2. 选择你的 GitHub 仓库
3. 点击 **"Import"**

### 3. 配置部署参数

| 配置项 | 值 |
|--------|------|
| Framework Preset | Next.js |
| Root Directory | frontend |
| Build Command | 保持默认 |
| Output Directory | 保持默认 |

### 4. 配置环境变量

在 **"Environment Variables"** 中添加：

| 变量名 | 值 | 说明 |
|--------|------|------|
| NEXT_PUBLIC_API_BASE | `https://your-backend-domain.onrender.com/api` | 后端公网地址 |

### 5. 部署

点击 **"Deploy"** 开始部署。

部署成功后，你将获得一个 Vercel 域名，格式类似：
`https://pet-care-platform-xxxx.vercel.app`

---

## 三、前后端联动配置

### 1. 获取后端地址

后端部署成功后，从 Render Dashboard 获取公网地址。

### 2. 更新后端 CORS 配置

在 Render 的环境变量中，更新 `ALLOWED_ORIGINS`：
```
https://pet-care-platform-xxxx.vercel.app
```

### 3. 更新前端 API 地址

在 Vercel 的环境变量中，更新 `NEXT_PUBLIC_API_BASE`：
```
https://pet-care-backend-xxxx.onrender.com/api
```

### 4. 重新部署

修改环境变量后，两个平台都会自动触发重新部署。

## 四、SQLite 在 Render 免费层的缺陷与解决方案

### 缺陷说明

Render 免费层的实例会在空闲 15 分钟后自动休眠，唤醒时会重新部署。由于 SQLite 数据库文件存储在容器内部的 `backend/instance/` 目录中，每次重新部署都会**清空所有数据**。

### 临时演示解决方案

对于演示目的，可以接受这个限制，但需要注意：

1. **数据会丢失**：每次实例休眠后重启，数据库会重置为初始状态
2. **首次访问较慢**：实例从休眠状态唤醒需要 30-60 秒

### 长期解决方案（如需持久化）

如果需要数据持久化，建议：

1. **升级到 Render 付费层**：$7/month 起，提供持久磁盘
2. **使用外部数据库**：
   - Render PostgreSQL（免费层可用）
   - Supabase（免费层可用）
   - 其他云数据库服务

#### 切换到 PostgreSQL（Render 免费层）

1. 在 Render 创建 PostgreSQL 数据库
2. 获取数据库连接字符串
3. 修改 `backend/.env`：
   ```
   DATABASE_URL=postgresql://user:password@host:port/database
   ```
4. 更新 `backend/app.py` 中的数据库配置

## 五、部署文件清单

### 后端配置文件
- `backend/render.yaml` - Render 部署配置
- `backend/requirements.txt` - Python 依赖清单（含 gunicorn）
- `backend/app.py` - 应用入口（生产环境适配）

### 前端配置文件
- `frontend/.env.local` - 本地开发环境变量
- `frontend/next.config.js` - Next.js 配置（含 API 代理）
- `frontend/lib/api.ts` - API 请求封装（读取环境变量）

---

## 六、本地开发

### 后端启动
```bash
cd backend
pip install -r requirements.txt
python app.py
```

### 前端启动
```bash
cd frontend
npm install
npm run dev
```

### 环境变量
- 后端：复制 `backend/.env.example` 为 `backend/.env`
- 前端：使用 `frontend/.env.local`（已配置本地 API 地址）
