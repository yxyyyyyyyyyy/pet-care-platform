# 调试日志

## 项目概述

宠物健康养护平台，基于 Next.js (App Router) + Flask + SQLite 全栈架构。

---

## 问题汇总

### 1. 前端 TypeScript 导入路径报错（红色波浪线）

**现象**：VS Code 中大量 tsx 文件存在红色波浪导入警告

**原因**：全局使用 `@/` 路径别名，但 tsconfig.json 和 next.config.js 配置不一致，VS Code 无法正确解析

**解决方案**：
- 移除所有 `@/` 路径别名，统一改为相对路径
- 删除 tsconfig.json 中的 paths 配置
- 删除 next.config.js 中的 webpack alias 配置

**涉及文件**：
- `frontend/app/login/page.tsx`
- `frontend/app/pets/page.tsx`
- `frontend/app/care/page.tsx`
- `frontend/components/Layout.tsx`
- `frontend/components/AuthGuard.tsx`
- `frontend/components/Modal.tsx`
- `frontend/components/Alert.tsx`
- `frontend/lib/api.ts`

---

### 2. React 类型未正确导入

**现象**：`React.ReactNode` 和 `React.FormEvent` 报错

**原因**：未从 react 包导入这些类型

**解决方案**：
- 从 react 导入 `type ReactNode` 和 `type FormEvent`
- 使用类型别名替代 `React.XXX` 写法

**涉及文件**：
- `frontend/components/Layout.tsx`
- `frontend/components/AuthGuard.tsx`
- `frontend/components/Modal.tsx`
- `frontend/app/login/page.tsx`
- `frontend/app/pets/page.tsx`
- `frontend/app/care/page.tsx`

---

### 3. NextRouter was not mounted 错误

**现象**：首页 `app/page.tsx` 运行时报错 "NextRouter was not mounted"

**原因**：错误使用 `next/router`（Pages Router API），而非 `next/navigation`（App Router API）

**解决方案**：
- 将所有 `useRouter` 导入从 `next/router` 改为 `next/navigation`

**涉及文件**：
- `frontend/app/page.tsx`
- `frontend/app/login/page.tsx`
- `frontend/components/Layout.tsx`
- `frontend/components/AuthGuard.tsx`

---

### 4. 后端 SQLite 数据库无法打开

**现象**：启动时报错 `sqlite3.OperationalError: unable to open database file`

**原因**：数据库路径配置错误，使用相对路径导致解析失败

**解决方案**：
- 使用绝对路径动态计算数据库位置
- 自动创建 instance 目录

**涉及文件**：
- `backend/app.py`

---

### 5. Flask 依赖缺失

**现象**：启动时报错找不到 Werkzeug

**原因**：requirements.txt 缺少 Werkzeug 依赖

**解决方案**：
- 在 requirements.txt 中添加 `Werkzeug==2.3.7`

**涉及文件**：
- `backend/requirements.txt`

---

### 6. CORS 预检请求报错 (preflight request Redirect is not allowed)

**现象**：浏览器控制台报 CORS 错误，OPTIONS 预检请求被重定向

**原因**：
1. Flask-CORS 配置缺少 `Authorization` 请求头支持
2. 路由末尾斜杠不一致导致 301 重定向
3. JWT 鉴权失败时重定向到登录页面

**解决方案**：
- 完善 Flask-CORS 配置，允许 `Authorization` 头和 OPTIONS 方法
- 统一路由定义，移除前导斜杠
- 配置 JWT 错误处理器，统一返回 JSON 格式

**涉及文件**：
- `backend/app.py`
- `backend/routes/auth.py`
- `backend/routes/pets.py`
- `backend/routes/care.py`

---

### 7. JWT 多实例冲突

**现象**：JWT 认证失败，即使携带正确 Token 也返回 401

**原因**：在 app.py 中重新创建了 JWTManager 实例，但路由文件仍从 extensions.py 导入另一个实例，导致 Token 验证使用的密钥不一致

**解决方案**：
- 统一从 `extensions.py` 导入 `jwt` 实例
- 使用 `jwt.init_app(app)` 初始化

**涉及文件**：
- `backend/app.py`
- `backend/extensions.py`

---

### 8. JWT Identity 类型问题

**现象**：Token 中的 identity 为整数，但 get_jwt_identity() 无法正确识别

**原因**：`create_access_token(identity=user.id)` 使用整数作为 identity，JWT 库期望字符串类型

**解决方案**：
- 创建 Token 时转换为字符串：`create_access_token(identity=str(user.id))`
- 使用时转换回整数：`user_id = int(get_jwt_identity())`

**涉及文件**：
- `backend/routes/auth.py`
- `backend/routes/pets.py`
- `backend/routes/care.py`

---

### 9. SQLite Date 类型转换错误

**现象**：新增宠物时，birthday 字符串直接保存导致报错 "SQLite Date type only accepts Python date objects"

**原因**：前端传递的是日期字符串，后端未转换为 Python date 对象直接存入数据库

**解决方案**：
- 使用 `date.fromisoformat()` 将字符串转换为 date 对象
- 添加判空检查，空值时赋值为 None

**涉及文件**：
- `backend/routes/pets.py`
